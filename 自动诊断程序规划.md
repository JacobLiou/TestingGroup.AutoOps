## 一、背景与目标

随着产线快速扩张，新人比例上升，系统与工位问题的定位成本显著增加。目前大量问题集中在：

- 系统侧基础环境、接口、配置缺失或错误
- 平台化工位在设备、光源、参数一致性上的隐性问题

目标是建设一套通用、自助、标准化的自诊断程序：

- 系统端自诊断：一次性覆盖大部分系统类问题，统一放到 MIMS 上运行
- 业务端/工位自诊断：基于 UDL 配置，快速判断设备与光学质量，减少现场排障依赖

最终效果：

- 新人可独立完成 70~80% 的问题初判
- 问题从“经验判断”转为“结果说话”
- 形成可持续扩展的诊断框架

## 三、系统端自诊断（MIMS）

负责人建议：彭娟 / 万新

设计目标：

- 把“系统问题”拆成可枚举、可执行、可判定的规则
- 每条规则都能明确回答三件事：

1）现在是否正常？2）哪里不正常？3）下一步该找谁/怎么处理？

### 3.1 规则总体模型（建议统一）

Rule 基本结构

- RuleCode：规则唯一标识（如 SYS-NET-001）
- RuleName：规则名称
- Category：分类（网络 / 接口 / 配置 / 权限 / 数据）
- CheckLogic：检测逻辑（程序实现）
- Threshold / Expectation：阈值或期望状态
- ResultLevel：OK / Warning / Error
- FailMessage：失败说明（给新人看）
- Suggestion：处理建议 / 升级路径

✅ 要求：任何系统诊断项，都必须能落到一条或多条 Rule 上

### 3.2 网络与基础环境诊断规则

#### 3.2.1 工程文件夹访问类

- SYS-NET-001 工程目录可达性检查
  - 检查项：核心工程/共享目录是否可访问
  - 逻辑：尝试建立连接
  - 结果：
  - OK：可访问
  - Error：无法访问
  - 常见原因：网络不通 / 路径错误
- SYS-NET-002 工程目录读权限检查
  - 逻辑：创建/读取临时文件
  - Error 说明：无读权限
  - 建议：检查账号权限 / AD 组
- SYS-NET-003 工程目录写权限检查
  - 逻辑：写入并删除临时文件
  - Warning：可读不可写
  - Error：完全无权限
- SYS-NET-004 访问性能检查
  - 指标：访问延迟 / 超时率
  - Warning：延迟超过阈值（如 >500ms）
  - 说明：可能影响生产节拍

#### 3.2.2 服务器连通性类

- SYS-NET-010 服务器 Ping 可达性
  - 检查对象：MES / PLM / 文件服务器等
  - Error：Ping 不通
- SYS-NET-011 端口可用性检查
  - 检查：TCP 端口是否监听
  - Error：端口未开放 / 被防火墙阻断
- SYS-NET-012 域/账号状态检查
  - 检查：账号是否被禁用 / 过期
  - Error：账号异常

### 3.3 系统接口健康诊断规则

#### 3.3.1 接口配置完整性

- SYS-API-001 接口地址配置检查
  - Error：接口地址为空或非法
- SYS-API-002 鉴权信息完整性检查
  - Error：Token / Key 缺失

#### 3.3.2 接口连通与响应

- SYS-API-010 接口连通性检查
  - 逻辑：发起轻量心跳请求
  - Error：请求失败（超时/拒绝）
- SYS-API-011 接口响应码检查
  - OK：200
  - Warning：非 200 但可识别
  - Error：鉴权失败 / 服务异常
- SYS-API-012 接口响应时间检查
  - Warning：响应时间 > 阈值（如 1s）

#### 3.3.3 接口数据正确性

- SYS-API-020 返回数据结构校验
  - Error：字段缺失 / 结构不匹配
- SYS-API-021 关键字段有效性检查
  - Error：ID / Code 为空或非法

### 3.4 模板与配置一致性规则

#### 3.4.1 模板存在性与版本

- SYS-CFG-001 工艺模板存在性检查
  - Error：模板缺失
- SYS-CFG-002 模板版本匹配检查
  - Warning：版本不一致

#### 3.4.2 模板内容完整性

- SYS-CFG-010 模板字段完整性检查
  - Error：必填字段缺失
- SYS-CFG-011 字段取值合法性检查
  - Warning：超范围 / 非法值

#### 3.4.3 系统间关联配置

- SYS-CFG-020 系统 ID / Code 映射检查
  - Error：映射缺失或冲突
- SYS-CFG-021 跨系统一致性检查
  - Warning：MES / PLM 数据不一致

### 3.5 运行环境与基础依赖（可选增强）

- SYS-ENV-001 本地环境版本检查
  - .NET / Runtime / 驱动版本
- SYS-ENV-002 时间同步检查
  - Warning：系统时间偏差过大
- SYS-ENV-003 磁盘空间检查
  - Warning：剩余空间不足

### 3.6 规则执行与结果汇总原则

- 所有 Rule 独立执行，不互相阻断
- 结果自动汇总为：
  - 系统整体状态（绿/黄/红）
  - 高优先级问题 Top N
- 每条 Error 必须绑定明确 Suggestion

✅ 最终目标：

看到诊断结果的人，不需要再“问系统怎么回事”

### 3.2 系统接口健康检查

覆盖接口（可配置扩展）：

- MES
- PLM
- TMS
- TAS
- 其他内部系统

检查维度：

- 接口地址是否配置
- 鉴权信息是否缺失/过期
- 心跳/轻量请求是否成功
- 返回数据结构是否符合预期

输出示例：

- ✅ MES 接口正常（200 / 响应 85ms）
- ⚠️ TMS 接口响应慢（>1s）
- ❌ PLM Token 已失效

### 3.3 模板与关联配置检查

- 工艺模板是否存在
- 版本是否匹配
- 模板字段完整性
- 系统间关联配置（ID / Code 映射）

重点解决：

“系统能打开，但一运行就报各种奇怪错误”

### 3.4 通用性设计原则

- 规则插件化：每一类检查一个 Rule

- 结果标准化

  ：

  - OK / Warning / Error

- 一次执行，全量覆盖

- 结果可沉淀（用于统计高频问题）

## 四、业务端 / 工位自诊断（平台化工位）

建议协作：Zhao Hanfei / Min Dong / Duan Yu

### 4.1 触发方式

- 工位上位机一键“自诊断”
- 新工位上线 / 设备更换后必跑
- 问题工位复现前先跑

### 4.2 基于 UDL 的自动识别

从 UDL 中自动获取：

- 设备型号
- 相机 / 光源类型
- 通道数量
- 标准参数范围

做到：

不靠人选配置，靠规则匹配

### 4.3 光源与成像质量诊断

统一诊断指标（可平台化）：

光源质量

- 最大功率
- 功率稳定性（时间维度波动）
- 开关重复性

成像质量

- 信噪比（SNR）
- 平坦度（均匀性）
- 灰度一致性

结果输出：

- 当前值
- 基线值 / 阈值
- 是否达标

### 4.4 设备与工位一致性检查

- 设备参数是否被私自修改
- 同类工位参数横向对比
- 当前工位 vs 标准工位差异

用于解决：

“这台怎么总是有问题，但又说不清”
